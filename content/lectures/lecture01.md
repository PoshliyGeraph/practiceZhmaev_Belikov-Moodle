** 1.1 От файловых систем к СУБД: историческая ретроспектива **
Эра файловых систем (1950-60-е)
В первых компьютерах данные хранились в плоских текстовых файлах. Каждая программа сама заботилась о формате и обработке.

Код на COBOL для чтения файла сотрудников:


DATA DIVISION.
FILE SECTION.
FD  EMPLOYEE-FILE.
01  EMPLOYEE-RECORD.
    05 EMP-ID        PIC 9(5).
    05 EMP-NAME      PIC X(30).
    05 EMP-SALARY    PIC 9(7)V99.
Проблемы файлового подхода:

Избыточность данных: одни и те же поля повторяются в разных файлах

Трудности параллельного доступа: два пользователя не могут одновременно редактировать файл

Отсутствие стандартов: каждая программа — свой формат

Нарушение целостности: нет гарантий, что данные не испортятся

Рождение СУБД (1970-е)
Эдгар Кодд из IBM публикует статью "A Relational Model of Data for Large Shared Data Banks" (1970), закладывая основы реляционного подхода.

1.2 Базовая терминология
База данных (БД) — поименованная совокупность структурированных данных, относящихся к определенной предметной области.

Система управления базами данных (СУБД) — комплекс программных средств, обеспечивающих создание, ведение и совместное использование БД.

Аналогия:

🏛️ БД — это склад с деталями (сами данные)

👨‍🔧 СУБД — это кладовщик со своей системой учета (программа, управляющая данными)

1.4 Реляционная модель данных
Определения
Отношение (Relation) — двумерная таблица

Кортеж (Tuple) — строка таблицы

Атрибут (Attribute) — столбец таблицы

Домен (Domain) — множество допустимых значений атрибута

Кардинальность — количество строк

Степень — количество атрибутов

Пример отношения "Студенты"
id (INT)	full_name (VARCHAR)	birth_date (DATE)	gpa (DECIMAL)
1	Иванов Иван	2003-05-12	4.5
2	Петрова Мария	2002-11-03	4.8
3	Сидоров Алексей	2003-08-21	3.9
Ключи
Первичный ключ (Primary Key) — атрибут, уникально идентифицирующий кортеж

Должен быть уникальным

Не может быть NULL

Может быть составным (несколько столбцов)

Внешний ключ (Foreign Key) — ссылка на первичный ключ другой таблицы

Обеспечивает связь между таблицами

Может быть NULL (если связь необязательна)

1.5 Знакомство с SQL
Типы данных (стандарт SQL)
sql
-- Числовые типы
INT, SMALLINT, BIGINT           -- целые числа
DECIMAL(10,2), NUMERIC          -- дробные с фиксированной точностью
FLOAT, REAL                     -- числа с плавающей точкой

-- Строковые типы
CHAR(n)                         -- фиксированная длина (дополняется пробелами)
VARCHAR(n)                      -- переменная длина (до n символов)
TEXT                            -- длинные тексты

-- Дата и время
DATE                            -- дата (год-месяц-день)
TIME                            -- время (часы:минуты:секунды)
TIMESTAMP                       -- дата + время
INTERVAL                        -- интервал времени

-- Логический тип
BOOLEAN                         -- TRUE, FALSE, NULL
Первая база данных: интернет-магазин
Создание таблицы товаров:

sql
CREATE TABLE products (
    id INT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    price DECIMAL(10,2) CHECK (price > 0),
    in_stock BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
Заполнение данными:

sql
INSERT INTO products (id, name, price, in_stock) VALUES
(1, 'Ноутбук', 75000.50, TRUE),
(2, 'Мышь', 1500.00, TRUE),
(3, 'Клавиатура', 3500.00, FALSE);
Простейшая выборка:

sql
SELECT * FROM products;
SELECT name, price FROM products WHERE in_stock = TRUE;
