Глоссарий по курсу «Базы данных»
Лекция 1: Фундамент. Введение и реляционная модель

СУБД  -   Система управления базами данных — комплекс программных средств, обеспечивающих создание, ведение и совместное использование баз данных
Кортеж  -  Строка таблицы (отношения), представляющая собой набор значений атрибутов одной сущности
Первичный ключ - Поле или набор полей, которые уникально идентифицируют каждую запись в таблице; не может содержать NULL
Внешний ключ   -  Поле в таблице, которое ссылается на первичный ключ другой таблицы для обеспечения связи между ними
Домен  -  Множество допустимых значений, которые может принимать атрибут (например, возраст от 0 до 150)

Лекция 2: Математика запросов. SELECT и операции
  
Проекция  - Операция реляционной алгебры, выбирающая указанные столбцы из отношения (аналог SELECT в SQL)
Селекция  - Операция реляционной алгебры, выбирающая строки, удовлетворяющие заданному условию (аналог WHERE в SQL)
Агрегация  - Процесс вычисления итоговых значений (сумма, среднее, количество) по набору строк
Кардинальность  -  Количество строк (кортежей) в таблице или результате запроса
Декартово произведение  -  Операция, комбинирующая каждую строку одной таблицы с каждой строкой другой таблицы
Лекция 3: Связи и целостность. Проектирование БД
  
Нормализация  - Процесс устранения избыточности и аномалий путем приведения таблиц к нормальным формам
ER-диаграмма  - Визуальная модель данных, отображающая сущности предметной области и связи между ними
Ссылочная целостность  - Требование, согласно которому внешний ключ должен ссылаться только на существующие записи
Денормализация  - Намеренное нарушение нормальных форм для повышения производительности (меньше JOIN'ов)
Каскадирование  - Автоматическое обновление или удаление связанных записей при изменении родительской записи
Лекция 4: Транзакции, индексы и оптимизация
  
Транзакция  - Логическая единица работы, которая выполняется полностью или не выполняется вовсе (свойство атомарности)
Индекс  - Структура данных (обычно B-Tree), ускоряющая поиск строк по определенным колонкам за счет дополнительного места
ACID  - Набор свойств транзакций: атомарность, согласованность, изолированность, надежность
Блокировка  - Механизм синхронизации доступа к данным при параллельной работе нескольких транзакций
План запроса  - Последовательность операций (сканирование таблиц, использование индексов, соединения), которую выбирает оптимизатор СУБД
Лекция 5: Администрирование и современные тренды
  
Репликация  - Процесс копирования данных с одного сервера на другие для повышения отказоустойчивости и распределения нагрузки
Шардинг  - Горизонтальное разделение данных по нескольким серверам на основе ключа (например, по ID пользователя)
NoSQL  - Общий термин для нереляционных баз данных, использующих другие модели (ключ-значение, документы, графы)
CAP-теорема  - Теорема, утверждающая, что распределенная система может обеспечить только два из трех свойств: согласованность, доступность, устойчивость к разделению
BASE  - Подход к транзакциям в NoSQL (базовая доступность, мягкое состояние, конечная согласованность) — альтернатива ACID
